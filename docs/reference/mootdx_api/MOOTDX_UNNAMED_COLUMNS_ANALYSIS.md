# Mootdx未命名列分析报告

## 核心发现

通过查看mootdx源码 `/mootdx/financial/columns.py`，我们发现：

**mootdx官方本身就没有给这182个列命名！**

```python
# columns.py 第323-560列
'每股股东自由现金流',
'col323',  # ← 从这里开始未命名
'col324',
'col325',
...
'col560',  # ← 到这里结束未命名
'加:其他原因对现金的影响2(万元)(现金的期末余额科目)',  # ← 后面又有命名
```

## 统计信息

- **总列数**: 585列
- **未命名列数**: 238列 (col323-col560)
- **未命名列占比**: 40.7%
- **位置范围**: 323-560 (连续238列)

## 未命名列的区域划分

根据数据特征，我们可以将未命名列大致分为几个区域：

### 区域1: col323-col329 (每股指标扩展区)

**位置**: 紧跟在"每股股东自由现金流"之后

| 列名 | 推测含义 | 数值特征 | 非零率 |
|------|---------|---------|--------|
| col323 | 企业自由现金流总额(元) | 均值11万，中位数1.1万 | 100% |
| col324 | 股东自由现金流总额(元) | 均值1.7万，中位数3363 | 75.7% |
| col325 | **是否分红标志位** | 只有0和1 | 17.6% |
| col326 | 分红金额(元) | col325=1时有值 | 17.6% |
| col327 | 市盈率或倍数 | 0-100范围 | 88.9% |
| col328 | 某种金额指标 | 均值26万 | 73.9% |
| col329 | 年限或百分比 | 0-30范围 | 92.6% |

**关键推断**:
- **col325是标志位**: 935只股票(17.6%)的值为1，可能代表"本期有分红"
- **col326是分红金额**: 只有col325=1的股票才有值，范围17万到15亿
- **col323/col324**: 根据上下文推测是现金流相关指标的金额形式

### 区域2: col330-col500 (未知扩展指标区)

**特征**:
- 大部分列非零率很低（<5%）
- 偶尔出现非零率较高的列（>80%）
- 数值量级差异很大

**可能用途**:
1. 特殊行业指标（如金融、保险、房地产）
2. 新会计准则增加的字段
3. 预留的扩展字段
4. 衍生计算指标

### 区域3: col501-col560 (预留区域)

**特征**:
- 绝大多数列非零率<1%
- 可能是预留字段

### 特殊发现: col560之后恢复命名

col560之后突然出现了命名字段（都是金融类特殊指标）：

```python
'加:其他原因对现金的影响2(万元)(现金的期末余额科目)',
'客户存款和同业存放款项净增加额(万元)',
'向中央银行借款净增加额(万元)',
'向其他金融机构拆入资金净增加额(万元)',
'收到原保险合同保费取得的现金(万元)',
...
```

这些都是**金融机构专属字段**（银行、保险、证券公司使用）。

## 为什么会有这么多未命名列？

### 1. 通达信数据格式的历史演进

通达信财务数据格式可能经历了多次扩展：
- **最初版本**: 只有基础的三大报表（资产负债表、利润表、现金流量表）
- **扩展阶段1**: 增加财务比率、机构持股等衍生指标
- **扩展阶段2**: 预留大量扩展字段以支持新会计准则
- **扩展阶段3**: 添加行业特殊字段（金融、保险等）

### 2. 字段映射不完整

可能的原因：
1. **数据提供方未公开字段定义**: 通达信从Wind、东方财富等获取原始数据时，可能只有数据文件没有字段说明
2. **字段定义文档丢失**: mootdx开发者可能也不知道这些列的确切含义
3. **商业保密**: 某些衍生指标的计算方法可能是商业机密

### 3. 数据质量参差不齐

部分列非零率极低（<1%），说明：
- 可能是特定行业才有的字段
- 可能是新增字段，历史数据为空
- 可能是废弃字段，现在已不再更新

## 推断未命名列含义的方法

### 方法1: 数值特征分析

```python
# 统计特征
非零率: 判断该列是否常用
数值范围: 推断单位（元、万元、百分比等）
数值分布: 推断业务含义（金额、比率、天数等）
```

### 方法2: 与已知列的相关性

```python
# 寻找计算关系
if col323 ≈ 每股企业自由现金流 * 总股本 * 单位转换:
    col323可能是"企业自由现金流总额"
```

### 方法3: 标志位识别

```python
# col325只有0和1
if col325 == 1:
    col326有值  # 说明col326是col325的关联数据
```

### 方法4: 上下文推断

根据前后列的名称推测业务含义。例如：
- col323紧跟"每股股东自由现金流" → 可能是现金流相关
- col560之后是金融类字段 → col560前可能是金融类预留字段

## 对数据处理的影响

### 1. 不能简单删除未命名列

虽然这些列没有名称，但它们包含真实数据：
- col323-col329有较高非零率（75%-100%）
- 这些数据对某些分析可能有价值

### 2. 建议的处理策略

#### 策略A: 保留所有列（当前实现）
```python
# 保持原样，不做改名
df = Affair.parse(downdir, filename)
```

**优点**: 不丢失任何信息
**缺点**: 列名无意义，用户难以使用

#### 策略B: 添加元数据标注
```python
# 为列添加描述性元数据
column_metadata = {
    'col323': {'推测': '企业自由现金流总额', '确定性': '高', '非零率': 1.0},
    'col324': {'推测': '股东自由现金流总额', '确定性': '高', '非零率': 0.757},
    'col330': {'推测': '未知', '确定性': '低', '非零率': 0.002},
}
```

#### 策略C: 分类存储
```python
# 将数据分为三类
df_named = df[named_columns]         # 有明确名称的列
df_inferred = df[inferred_columns]   # 可推断含义的列
df_unknown = df[unknown_columns]     # 完全未知的列
```

## 结论

mootdx的未命名列不是bug，而是**数据格式设计的遗留问题**：

1. **官方也不知道**: mootdx开发者自己也没有这些列的文档
2. **数据是真实的**: 这些列包含真实财务数据，只是缺少字段说明
3. **推断有一定可行性**: 通过数据特征分析，我们可以推断部分列的含义
4. **完全准确需要官方文档**: 最终还是需要通达信官方提供完整的字段定义
