## 代码使用示例

### 示例1: 获取无重复的列名

```python
from mootdx.affair import Affair

df = Affair.parse(downdir='./data', filename='gpcw20191231.zip')

# 原始列名有重复，pandas会警告
# 使用simtradedata的适配器自动处理
from simtradedata.data_sources.mootdx_adapter import MootdxAdapter

adapter = MootdxAdapter()
adapter.connect()

# 获取财务数据（自动去重）
result = adapter.batch_import_financial_data('2019-12-31', 'Q4')
df_clean = pd.DataFrame(result['data'])
```

### 示例2: 使用推断的列名

```python
# 启用高确定性列名推断
df_inferred = adapter._infer_column_names(df, confidence_level='high')

# 现在可以使用有意义的列名
分红股票 = df_inferred[df_inferred['是否分红标志(0否1是)'] == 1]
平均分红 = 分红股票['分红金额(元)'].mean()
```

### 示例3: 处理重复列名

```python
# 获取"财务费用"的两个版本
财务费用_利润表 = df['财务费用']        # 第1次出现（位置80）
```

### 示例4: 查询高质量列

```python
# 只使用非零率>50%的列（质量较高）
high_quality_cols = []

for col in df.columns:
    non_zero_rate = (df[col] != 0).sum() / len(df)
    if non_zero_rate > 0.5:
        high_quality_cols.append(col)

df_clean = df[high_quality_cols]
```

## 常见陷阱

### ⚠️ 陷阱1: 误用重复列

```python
# ❌ 错误：直接用列名，可能拿到错误的版本
净利润 = df['归属于母公司所有者的净利润']  # 不确定是哪个版本

# ✅ 正确：明确指定需要的版本
净利润_年度 = df.iloc[:, 96]  # 位置96：年度累计
净利润_单季 = df.iloc[:, 232]  # 位置232：单季度
```

### ⚠️ 陷阱2: 忽略列的口径差异

```python
# ❌ 错误：假设所有"净利润"都一样
净利润1 = df.iloc[:, 96]   # 归属母公司（年度）
净利润2 = df.iloc[:, 232]  # 归属母公司（单季度）
assert (净利润1 == 净利润2).all()  # ❌ 会失败！

# ✅ 正确：理解不同口径
年度净利润 = df.iloc[:, 96]
单季度净利润 = df.iloc[:, 232]
# 两者不能直接比较
```

### ⚠️ 陷阱3: 使用低质量未命名列

```python
# ❌ 错误：使用非零率极低的列
神秘指标 = df['col330']  # 非零率0.2%，99.8%都是0

# ✅ 正确：先检查数据质量
non_zero_rate = (df['col330'] != 0).sum() / len(df)
if non_zero_rate > 0.5:
    指标值 = df['col330']
else:
    print(f"警告：col330非零率仅{non_zero_rate:.1%}，不建议使用")
```

## 快速诊断清单

使用mootdx财务数据前，检查：

- [ ] 是否有重复列名？→ 使用`_deduplicate_columns()`
- [ ] 是否需要推断列名？→ 使用`_infer_column_names()`
- [ ] 是否理解重复列的差异？→ 查看本文档"重复列名速查表"
- [ ] 列的非零率是否>50%？→ 低质量列不推荐使用
- [ ] 是否选择了正确的口径？→ 年度vs单季度，权责制vs收付制
